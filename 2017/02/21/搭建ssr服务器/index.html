搭建ssr服务器配置环境

2017.02.21 Cj  热度 ℃

如果没有国外vps的需要先购买：https://virmach.com/manage/clientarea.php?action=services
购买完vps账号后，利用SecureCRT软件连接服务器(推荐服务器系统:debian7)

接下来就是ssr的搭建：
1.首先安装更新
apt-get update
2.安装git
apt-get install git
3.获取源代码
git clone -b manyuser https://github.com/shadowsocksr/shadowsocksr.git
4.在服务端配置
cd ~/shadowsocksr
bash initcfg.sh
进入子目录
cd ~/shadowsocksr/shadowsocks

接下来运行的方法有两种，一种是快速运行，一种是配置文件运行（推荐）
1）.快速运行
python server.py -p 443 -k password -m aes-256-cfb -O auth_sha1_v4 -o http_simple
在這里可能会发现一个问题,出现python：command not found
解决该命令方法:执行安装python模块命令
apt-get install -y python-pip

继续，如果要后台运行
python server.py -p 443 -k password -m aes-256-cfb -O auth_sha1_v4 -o http_simple -d start

查看日志
tail -f /var/log/shadowsocks.log

如果要停止/重启：
python server.py -d stop/restart

使用配置文件运行

如果你的ss目录是~/shadowsocksr，进入这里
修改user-config.json中的server_port，password等字段，具体可参见：
https://github.com/breakwa11/shadowsocks-rss/wiki/config.json

运行子目录内的server.py：
python server.py

如果要在后台运行：
python server.py -d start

如果要停止/重启：
python server.py -d stop/restart

查看日志：
tail -f /var/log/shadowsocks.log

如果配置好文件后出现: common.py:236 unsupported addrtype 69, maybe wrong password or encryption method 后面balabala一大堆的错误
更改user-config.json里面的”protocol”: “auth_sha1_compatible”及”obfs”: “http_simple_compatible”

配置开机自启动
https://github.com/breakwa11/shadowsocks-rss/wiki/System-startup-script
配置开机自启动的时候，或许你又会出现一个错误,在执行service shadowsocks start的时候，显示shadowsocks： No such file or directory 的错误
运行cat -v /etc/init.d/shadowcocks进行诊断，你会发现每个换行后面都出现^M,这就是罪魁祸首
